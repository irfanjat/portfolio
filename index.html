01 About

A bit about me

I'm a CS student from Karachi who started learning DevOps in July 2025 and got completely hooked. Not because of trends — I genuinely liked the idea that infrastructure could be code, that deployments could be automatic, and that a single git push could do everything.

I started with tutorials, hit their limits fast, and moved to building real things. Most of what I know came from things breaking — a pod stuck in CrashLoopBackOff, an ALB that wouldn't provision because subnets weren't tagged, a pipeline that silently failed on push. That kind of learning sticks.

In 8 months I've built a full GitOps CI/CD pipeline, a modular Terraform AWS infrastructure, a Kubernetes observability stack, deployed on AWS EKS, and more. All 20+ projects are public on GitHub with proper READMEs and architecture docs — the same standard you'd expect from a real engineering team.

Open to remote, onsite, hybrid, and contract work. Available across time zones from Karachi.

What I've built

⭐ Featured Project

End-to-End GitOps CI/CD Pipeline

ProblemDeployments needed 8+ manual commands, took ~15 minutes, had no traceability or rollback. Any mistake caused downtime.
SolutionTwo-repo GitOps workflow — app repo triggers CI, config repo is the cluster source of truth, ArgoCD watches and auto-syncs.
Architecturegit push → GitHub Actions (pytest 11s → Docker build 81s → GHCR push → SHA tag) → ArgoCD detects change → rolling deploy on Kubernetes with readiness probes.
ToolsGitHub Actions · ArgoCD · Kubernetes · Docker · GHCR · Flask · pytest · Helm
OutcomeDeployment time cut from 15 min → under 2 min. Zero manual steps. Every deploy traceable to an exact commit SHA. ArgoCD self-heals any manual drift.

Multi-Tier AWS Infrastructure — Terraform

Observability

Kubernetes Observability Stack

ProblemNo visibility into cluster health. Pod failures only noticed when users complained. No central log aggregation.
SolutionFull stack via Helm: Prometheus scraping 14 targets, Grafana with a custom 6-panel dashboard, Loki for log aggregation, Alertmanager with 3 alert rules.
ArchitecturePromtail → Loki. Node Exporter + kube-state-metrics → Prometheus → Grafana. PrometheusRule CRDs: pod restarts, count < 2, memory > 100MB.
ToolsPrometheus · Grafana · Loki · Alertmanager · Promtail · Helm · Kubernetes
OutcomeAll 3 alerts validated through simulated failures. Logs and metrics in one Grafana view. Zero kubectl logs needed for routine debugging.

AWS EKS

Kubernetes Deployment on AWS EKS

ProblemNeeded to understand how Kubernetes primitives map to real AWS infrastructure — from cluster provisioning to external traffic.
SolutionFull EKS lifecycle: eksctl cluster + managed node group, IAM OIDC, AWS Load Balancer Controller via Helm, ALB in IP target mode.
ArchitectureInternet → AWS ALB → K8s Ingress → ClusterIP Service → Pods. IP mode bypasses node ports. Rolling updates with maxUnavailable=0.
ToolsAWS EKS · ALB Controller · eksctl · Kubernetes · Helm · IAM/OIDC · Docker
OutcomeApp live on EKS with zero-downtime rolling deploys. Documented and fixed 4 real operational issues encountered during setup.
