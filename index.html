01 About

Who I Am

I'm a CS student from Karachi who got genuinely hooked on DevOps in August 2025 — not because it was trending, but because I found something satisfying about making things actually work reliably. The idea that a single commit can trigger a tested, containerized, and monitored deployment with zero manual steps just clicked for me.

I started by following tutorials, hit their limits fast, and switched to building things that break in interesting ways. Debugging a CrashLoopBackOff at 1am, hunting down why an ALB wasn't provisioning because subnets weren't tagged — that's where I've actually learned. The frustrating parts stuck.

In roughly 8 months I've built a GitOps pipeline, a full AWS 3-tier infra in Terraform, a Kubernetes observability stack, and a bunch of smaller things — not to pad a resume, but because each one taught me something the previous didn't. 20+ public repos with real READMEs and documented architecture decisions.

I'm still early, I know that. But I'm the kind of person who reads the why behind a Helm chart before running it. Looking for a team where I can contribute and keep growing — remote, onsite, hybrid, or contract, based in Karachi.

What I've Built

⭐ Featured Project

End-to-End GitOps CI/CD Pipeline

A production-grade GitOps workflow where a single git push is the only manual step. Code goes from laptop to live Kubernetes deployment automatically — no manual kubectl apply, ever.

  • Two-repo GitOps pattern: app source repo + config repo as single source of truth for cluster state
  • Cut deployment time from 15 min to under 2 min using Docker layer caching and parallel Actions jobs
  • Immutable image tags with full commit-SHA traceability + instant rollback capability
  • ArgoCD self-heal detects and reverts any manual kubectl drift within minutes
  • Health-gated rolling updates — pods only receive traffic after passing readiness probes
GitHub Actions ArgoCD Kubernetes Docker GHCR Flask pytest Helm

git push → triggers CI

├─ detects new SHA commit
└─ rolling deploy → k8s

Kubernetes → 2 replicas
├─ readiness probes
├─ zero downtime
└─ self-heal enabled
